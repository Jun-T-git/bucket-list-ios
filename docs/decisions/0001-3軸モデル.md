# 0001. 時間階層を廃止し、独立3軸（優先度×季節×タグ）へ

- Status: Accepted
- Date: 2026-06（v2 設計時）

## Context
v1 は「今月 / 今年 / いつか」という時間階層で項目を分類していた。しかし時間で仕分けると、
「いつか」が肥大化してただの物置になり、消化（コアコンセプトの目的）に繋がらなかった。
また時間は「いつ差し出すか」の問題であって、項目の属性ではない。

## Decision
時間階層を IA から外し、独立した3軸でゆるく分類する：
- **優先度**：高(top)/中(maybe)/低(someday) — 1アイテム1つ
- **季節**：春/夏/秋/冬 ＋ いつでも — 複数選択可（**月単位指定は後に廃止**。commit 8da8eed）
- **タグ**：固定4個（食/旅/遊び/買い物）＋カスタム最大10 — 複数選択可

時間ウィンドウは「タイミング提案」（コアコンセプト§5③）へ寄せた。

## Consequences
- 「いつか」の物置化を回避。分類が属性ベースで安定。
- 季節は `SeasonTag`（`.season`/`.any`、空=いつでも）として実装。旧・月キー("m4"等)は季節へ畳み込み後方互換。
- タグは「地獄にしない・シンプル優先」を維持（固定4＋カスタム10上限）。
- 詳細実装：[data-model](../architecture/data-model.md)。
