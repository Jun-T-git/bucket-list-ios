# 0002. AI はオンデバイス完結・バックエンドを持たない

- Status: Accepted
- Date: 2026-06-27

## Context
URL やタイトルから「やりたいこと」を下ごしらえするのに AI が有効。だが 20代ユーザーの
「自分のリスト」というプライベート性と、運用コスト/依存を考えると、クラウド送信は避けたい。
コアコンセプト§7 も「AIは主役ではない。補助。依存しすぎない設計」と定めている。

## Decision
AI 処理はすべて**端末内**で完結し、バックエンドを持たない：
- URL からの候補生成は **Apple Foundation Models**（iOS 26 のオンデバイス LLM）。
- 入力に使うのは端末内取得メタデータ（LinkPresentation 中心）のみ。ページ本文・ユーザーテキストは外部送信しない。
- タイトル入力中の即時下書きは軽量な `Classifier`（正規表現）。
- Foundation Models 非対応端末（iOS<26 / 無効 / 非対応）は**ルールベースへフォールバック**。

## Consequences
- プライバシーと"自分のリスト"感を維持。サーバー運用コスト0。
- **ただし**：リンクのメタ取得自体は対象URLへの通信が発生する。よって「完全オフライン/一切通信しない」
  とは**謳わない**。正しい表現は「**推論はオンデバイス**」（[release-runbook](../workflows/release-runbook.md) 注記）。
- 多くのシミュレータは LLM 非対応 → ルールベース経路が常に成立することが必須（[capture-pipeline](../architecture/capture-pipeline.md)）。
- [設計原則§8](../philosophy/02-設計原則.md) として恒久化。
