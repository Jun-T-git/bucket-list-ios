# 意思決定ログ（ADR）— Wishes

ADR = Architecture Decision Record。**「なぜそう決めたか」を1決定1ファイルで固定**し、
将来の自分や別のAIが**同じ議論を蒸し返さない**ための記録。コードやコミットからは読み取れない
「理由」を残すのが目的。

## 使い方

- 大きな設計・方針の決定をしたら、`NNNN-短いタイトル.md` を追加（連番）。
- フォーマット（下記テンプレ）：**Status / Context / Decision / Consequences**。
- 既存の決定を覆す場合は、新しい ADR を追加し、古い方の Status を `Superseded by NNNN` にする。
- **[設計原則](../philosophy/02-設計原則.md)そのものを変えるときは、必ず対応する ADR を先に起票**する。

## テンプレート

```markdown
# NNNN. タイトル

- Status: Accepted | Superseded by NNNN | Deprecated
- Date: YYYY-MM-DD

## Context
（何が問題で、どんな制約・選択肢があったか）

## Decision
（何を選んだか）

## Consequences
（結果として何が良くなり、何を諦めたか。守るべき不変則）
```
