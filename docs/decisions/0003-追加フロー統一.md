# 0003. 追加フローを統一（AI/手入力モードを分けない）

- Status: Accepted
- Date: 2026-06〜07

## Context
初期案では「AIで作る」「自分で書く」を別モードにしていた。しかしモード選択は入口で迷いを生み、
「ユーザーが主役、AIは黙って補助」という思想（コアコンセプト§7）と噛み合わなかった。

## Decision
追加フローを1本に統一する：
- モード選択画面を作らない。**URL の有無で自然に振る舞いが決まる**。
- 全経路（アプリ内 `AddEditSheet`／共有 `ShareComposeView`）が**同一の `ItemForm`** を使う。
- AI は**ユーザーが未編集（untouched）のフィールドだけ**を補完する。触れた値は上書きしない。

## Consequences
- 入口の迷いが消え、手入力と AI 補助が地続きになった。
- `ItemForm` は store/Storage 非依存の純プレゼンテーションにして、アプリと拡張で共有（新規フォームを作らない）。
- 呼び出し側は "touched" 追跡でバインディングを包み、クローム と AI 採用の挙動だけを変える。
- [設計原則§5](../philosophy/02-設計原則.md) として恒久化。
